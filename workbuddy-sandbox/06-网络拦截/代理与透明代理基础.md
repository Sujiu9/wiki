# 🟡 代理与透明代理基础

> 📋 前置知识：理解 TCP 连接、HTTP 请求、host/port。

## 1. 为什么网络拦截先从代理学起

想拦截网络，最容易写出来的不是系统扩展，而是本地代理。代理模型能让你亲手实现：

- 解析目标地址；
- 判断 allow/deny；
- 放行时转发；
- 拒绝时返回错误；
- 记录 `NetworkBlockRecord`。

这比直接研究 NetworkExtension 更适合作为第一步。

## 2. 解决方案概览

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| HTTP Proxy | 客户端显式把请求发给代理 | HTTP 请求 |
| CONNECT Proxy | 客户端请求代理建立 TCP 隧道 | HTTPS |
| SOCKS Proxy | 通用 TCP 代理 | 多协议 |
| 透明代理 | 客户端无感，系统转发流量 | 全局拦截 |

WorkBuddySandbox 的 `LocalProxy` 是降级方案：当 macOS NE 未就绪时，通过代理环境变量尽量接管支持代理的程序。

## 3. 方案对比

| 维度 | 显式代理 | 透明代理 |
|------|----------|----------|
| 客户端是否感知 | 感知，需要配置 | 不感知 |
| 实现成本 | 低 | 高 |
| 覆盖范围 | 取决于程序是否尊重代理 | 更广 |
| 归因能力 | 可从进程环境推断 | 可结合 PID/规则 |

## 4. 如何写一个最小 CONNECT 代理

CONNECT 请求长这样：

```http
CONNECT example.com:443 HTTP/1.1
Host: example.com:443
```

最小处理流程：

```text
accept client
read first line
parse host:port
if denied(host, port):
  write "HTTP/1.1 403 Forbidden"
  record block
else:
  connect upstream
  write "HTTP/1.1 200 Connection Established"
  copy bytes both directions
```

伪代码：

```python
host, port = parse_connect(line)
if rules.deny(host, port):
    client.write(b"HTTP/1.1 403 Forbidden\r\n\r\n")
    record({"host": host, "port": port, "source": "local_proxy"})
else:
    upstream = tcp_connect(host, port)
    tunnel(client, upstream)
```

## 5. 常见坑

- **DNS 归因**：CONNECT 里有 host，但普通 TCP 可能只有 IP。
- **HTTPS 内容不可见**：代理只知道目标 host/port，不应解密用户流量。
- **程序不走代理**：很多底层库可能忽略 `HTTP_PROXY`。
- **循环代理**：代理自身发出的连接不能再次走自己。
- **阻塞转发**：双向 copy 要处理任一方向关闭。

## 7. 练习题：你要亲手实现什么

1. 写一个只支持 CONNECT 的本地代理。
2. 加入 deny 规则：拒绝 `example.com:443`。
3. 返回 403，并记录 `{host,port,scheme,source}`。
4. 给子进程注入 `HTTPS_PROXY=http://127.0.0.1:<port>`，验证 curl 是否经过代理。
5. 思考：如果程序直接用 socket 连接 IP，代理还能看到吗？
